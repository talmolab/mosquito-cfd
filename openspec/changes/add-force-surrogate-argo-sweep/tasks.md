A fake `mpi_runner(argv, *, cwd)` records the argv and writes a synthetic `IB_Particle_1.csv` of N rows into `cwd` (mirror the PR3 fake-executor) against a fixture manifest/deck in `tmp_path`:
   - **Completed** (scenario *Completed run writes the checked output and provenance*): full-length CSV → `run_config` returns `status=="completed"` (the reused `RunOutcome.status`, not a new dataclass); the CSV passes `check_completion`; `run.log` holds the runner output; `run_metadata.json` records `docker_image` (the `sha256:` digest), `deck_sha256` (assert **not None** — deck present), `timestamp` verbatim, `command` = `["mpirun","--allow-run-as-root","-np","1",<iamrex_binary>,"/workspace/<input_file>"]`, and the `extra_provenance` (workflow uid / pod / node / retry).
   - **Wing geometry staged** (scenario *Wing geometry is staged into the run directory*): after `run_config`, `<output-root>/<name>/wing.vertex` exists and byte-equals the source `wing_vertex` — the CFD deck resolves its geometry relative to the run dir, so a missing stage would be a silent solver failure.
   - **Exit code is exact** (scenario *Incomplete run signals retry*): `main([…full…], mpi_runner=<fake-full>) == 0`; `main([…short…], mpi_runner=<fake-short>) == 1`; `main([…], mpi_runner=<fake-nonzero>) == 1`. The fake is **injected via `main(argv, *, mpi_runner=None)`** (so `main` is testable cluster-free — it must NOT hardcode the real subprocess runner).
   - **Runner raises** (scenario *A raised runner is a clean failure*): a fake `mpi_runner` that raises `OSError` → `status=="failed"`, `run_metadata.json` written (`status="failed"`), the exception text (`repr(exc)`) is asserted **present in `run.log` file content**, `main()==1` — a clean pod failure, not a traceback.
   - **Malformed config fails fast** (scenario *Malformed configuration fails fast*): a non-positive `max_step` (or empty `input_file`/`name`) raises a config-named `ValueError` (assert the fake's call-count is **0** — no executor call); on the CLI, `main([...,'--max-step','0',...], mpi_runner=fake)` exits non-zero and a missing/non-integer `--max-step` is rejected by argparse (exit ≠ 0) — don't retry a hopeless config 5× on the A40.
   - **Native hardware (no override path)** (scenario *Native compute hardware*): assert `"source" not in meta["hardware"]` and `"workspace" not in meta["hardware"]` and `"hostname" in meta["hardware"]` (the base `get_hardware_info` dict — robust to CI `gpus:[]` *and* a dev GPU; **never** assert `gpus==[]`). **Structurally** assert (AST + `inspect.signature`) that `run_one_config` does **not** import `capture_compute_hardware`/`build_probe_command` and `run_config` has no `hardware`/`compute_hardware`/`executor` parameter — proving native-not-overridden cluster-free.
   - **Positive reuse** (enforces reuse, not reimplementation): an AST/import assertion that `run_one_config` **imports `check_completion`, `capture_surrogate_run_metadata`, `_format_run_log` from `mosquito_cfd.force_surrogate.runner`/`.sidecar`** (the positive mirror of the negative AST check above), **plus** a `monkeypatch` spy asserting `runner.check_completion` **is invoked** by `run_config` (required, not optional — the import check alone can't catch a copy-paste that leaves a dead import) — so the logic is the PR3 library, not a re-implementation.
   - **Retry overwrites in place** (D6, scenario underlies *Incomplete run signals retry*): calling `run_config` twice on the same run dir (simulating an Argo retry — first a short/failed CSV, then a full one) leaves exactly **one** `run_metadata.json`/`run.log` reflecting the **last** attempt (last write wins — no `*.1` duplication, no append).
   - **Mutable tag rejected** (scenario *Mutable tag rejected*): a tag without `sha256:` raises `ValueError`.
   - **Provenance is portable** (scenario *Provenance is portable*): reuse `_walk_strings`/`_assert_portable` — no `^[A-Za-z]:[\\/]` / `/hpi/`; deck recorded manifest-relative; command uses `/workspace/...`.
   - **`extra_provenance=None` default**: `run_config(..., extra_provenance=None)` writes valid metadata without the Argo keys and raises no `KeyError`/`TypeError` (the `main`-without-Argo-flags path).
   - **`main` argparse → provenance plumbing**: `main(["--workflow-uid","u","--pod","p","--node","n","--retry","2", …], mpi_runner=fake)` → the written `run_metadata.json` records those exact Argo values — proves the argparse→`run_config` glue.
   - **`container_workspace` is honoured (not hardcoded)**: with a non-default `--container-workspace`, the recorded `command` deck path uses that prefix.
   - **Re-export**: `hasattr(mosquito_cfd.force_surrogate, "run_config")` (mirrors PR3).
