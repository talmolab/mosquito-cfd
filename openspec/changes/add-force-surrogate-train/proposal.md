## Why

The Track B force-surrogate program ([docs/force_surrogate/roadmap.md](../../../docs/force_surrogate/roadmap.md), row #5; GitHub issue [#21](https://github.com/talmolab/mosquito-cfd/issues/21)) needs the **trained surrogate** at the center of the NVIDIA *Evidence-of-Readiness* figure: a small **kinematics(+phase) ‚Üí aerodynamic-force-coefficient** model, evaluated on **held-out kinematic configurations**, that proves the data‚Üísurrogate‚Üíinference pipeline runs end-to-end **on local hardware** and that the **NVIDIA PhysicsNeMo** stack is usable. PR4 (`add-force-surrogate-dataset`, #7) published the dataset; the PR #16 Argo corpus + PR #18 committed `examples/prelim_sweep/dataset.parquet` (109,656 rows, 27 configs, 21 train / 6 holdout **configs**), `dataset.units.json`, and `run_metadata.json`. What is still missing is the **trainer** that learns the map and emits the predictions + `metrics.json` that **PR6** (`add-force-surrogate-evidence-figure`) plots. Without it there is no prediction to plot and no readiness evidence.

This is the deliberately **easy, force-only** surrogate (design ¬ß9): an instantaneous **kinematics(+phase) ‚Üí force-coefficient** regression. It is **not** the funded surrogate ‚Äî **no** DoMINO encoder, **no** latent dynamics, **no** velocity/pressure field or plotfile reading, **no** RL-in-loop (CC-6). Those are H100-award deliverables and stay out of scope. **Training runs on the local RTX A5000** (24 GB, FP32/TF32) via WSL2 + `uv` ‚Äî *not* RunAI (RunAI is the A40 FP64-CFD path; offloading training there would undercut the *local-readiness* evidence and add cluster infra for a seconds-to-minutes job).

## What Changes

### Extends capability: `force-surrogate`

- **New module `src/mosquito_cfd/force_surrogate/train.py`** ‚Äî all logic; pure data/feature/split/metrics helpers import **no** torch (so the module imports cleanly in CI without the heavy `train` group, decision D2), while the model + training functions import `torch`/`physicsnemo` **lazily** inside themselves (GPU tier only):
  - `build_features(df) -> (X, feature_names)` ‚Äî input matrix `[stroke_amp_deg, frequency_fstar, pitch_amp_deg, sin(2œÄ¬∑phase), cos(2œÄ¬∑phase)]` (cyclic phase respects the 0/1 wrap; **no Reynolds** ‚Äî it is a deterministic function of `(œÜ, f*)` under the ŒΩ\*-fixed policy, so it carries no independent information, decision D3). `numpy`/`pandas` only.
  - `filter_converged_beat(df) -> df` ‚Äî keeps `wingbeat ‚â• 1` (the converged beat; the startup transient is dropped here **visibly**, not silently ‚Äî PR4 D3 carried it as an explicit tag for exactly this).
  - `make_config_splits(df, *, seed) -> SplitAssignment` ‚Äî the 6 `split == "holdout"` **configs** are the **test** set (CC-4, never touched for selection); a seeded **config-level** validation set is carved from the 21 train configs; train/val/test config sets are mutually disjoint.
  - `Standardizer` ‚Äî fits feature/target mean¬∑std on **train rows only** (no val/holdout leakage); transforms/inverse-transforms. `numpy` only.
  - `compute_metrics(y_true, y_pred, target_names) -> dict` ‚Äî per-target RMSE/MAE/R¬≤ + aggregate (numpy only, no torch).
  - `build_model(n_in, n_out, *, seed)` / `train_model(...)` / `predict(...)` ‚Äî the **PhysicsNeMo** regressor (its fully-connected/MLP architecture) and a seeded training loop; **lazy** torch/physicsnemo import. **PhysicsNeMo-only ‚Äî no PyTorch fallback model** (decision D1).
  - `evaluate_and_emit(...)` ‚Äî runs holdout inference, assembles `metrics.json` (per-target + aggregate + per-config + inference latency/throughput), the holdout-predictions frame, and the `run_metadata.json` via `capture_surrogate_run_metadata`.
- **Thin driver `scripts/train_surrogate.py`** ‚Äî CLI over the library (`--dataset`, `--out-dir`, `--seed`, `--docker-digest`, `--timestamp`, `--epochs`, `--wandb {disabled,online}`); resolves paths, calls the library, writes the four committed artifacts. No logic.
- **`__init__.py`** re-exports the new public API.

### Resolved decisions (see `design.md`)

- **D1 ‚Äî PhysicsNeMo-first, *no* PyTorch fallback.** The roadmap row lists a PyTorch fallback; **this change drops it** ‚Äî PhysicsNeMo is the only model. Track B's purpose is to demonstrate the PhysicsNeMo stack is usable locally; a PyTorch shadow model would dilute that evidence. De-risked by **task ordering**: the first task stands up PhysicsNeMo end-to-end (a GPU test) *before* the full pipeline, so an environment problem surfaces immediately with slack before the ~Jun-24 cutoff. A thin `build_model` seam is kept (good design), but only the PhysicsNeMo implementation ships. **Track B never gates submission** ‚Äî if PhysicsNeMo hard-blocks past the cutoff, the figure is cut and the proposal stands on the existing benchmarks.
- **D2 ‚Äî Lazy torch import; two-tier tests.** Pure helpers (features, filter, splits, standardizer, metrics) are torch-free and **gate CI**; the model/training functions import torch/physicsnemo lazily and are exercised only by **GPU tests** behind a registered `@pytest.mark.gpu` marker that auto-skips when CUDA is unavailable or `physicsnemo` is not importable. CI also runs `-m "not gpu"` (belt-and-suspenders, CC-2).
- **D3 ‚Äî Inputs: 3 kinematic knobs + cyclic phase; no Reynolds.** Standardized on **train rows only**.
- **D4 ‚Äî Targets: all six coefficients** `CF_x, CF_y, CF_z, CF_mx, CF_my, CF_mz` (multi-output). Keeps PR6 free to designate the single headline `CF_m` pitch axis (PR4 D2 deferred that pick). Targets standardized on train rows only and inverse-transformed before metrics/predictions.
- **D5 ‚Äî `uv` packaging; env is an operator deployment detail.** `nvidia-physicsnemo`, CUDA `torch`, and `wandb` go in an **optional `train` dependency-group** (not installed by default; `uv sync --group train`). The training code is env-agnostic: **WSL2 + uv** is the default, the **`nvcr.io/nvidia/physicsnemo` container** is the drop-in escape hatch, and a **native Linux workstation** needs zero code changes. **`uv.lock` is regenerated and committed with the `pyproject.toml` change in the same commit** ‚Äî `uv lock --check` (the staleness gate; *not* `uv sync --frozen`, which installs from the lock as-is without re-validating) requires the lock to represent the new group, the operator's `uv sync --group train` needs the group *in* the lock to resolve, and a future `uv lock`/`--locked` would otherwise diverge. The heavy deps are **marker-gated** (`; sys_platform == 'linux'`) so the cross-platform lock resolves on the Windows dev host while the wheels install only on the Linux training env; the de-risk task adds a pinned PyTorch CUDA index under the same gate if the default wheel lacks the build. Resolved `torch`/`physicsnemo` versions are captured in `run_metadata.json`.
- **D6 ‚Äî Reproducibility honesty (CC-1).** Explicit seeds + `torch.use_deterministic_algorithms`. Two determinism tiers: the **torch-free CPU helper chain** (features/standardizer/split/metrics) is **bitwise-reproducible and CI-tested**; the **seeded `train_model` step** is determinism-tested on the **CPU torch device in the torch tier** (needs torch, deselected in CI); the **CUDA/GPU run is seeded but not guaranteed bitwise** (cuDNN/TF32). `metrics.json` records `reproducibility.bitwise == "cpu_only"` ‚Äî documented, not overclaimed.
- **D7 ‚Äî wandb gating.** `wandb` logs the real A5000 runs (operator `--wandb online`); it defaults to **`disabled`** in CI and reruns (runners have no login). `metrics.json` is the committed source of truth, never a wandb network call.
- **D8 ‚Äî Committed artifacts.** `examples/prelim_sweep/metrics.json`, a holdout-**predictions** parquet, the trained **model checkpoint**, and `run_metadata.json` ‚Äî produced by the operator's A5000 run and committed (the dataset itself was likewise committed once the real corpus existed).

### Output artifacts

- **`metrics.json`** ‚Äî `per_target` (RMSE/MAE/R¬≤ for each of the six coefficients), `aggregate`, `per_config` (the 6 holdout configs), `inference` (latency/throughput for PR6's >1,000√ó annotation), and a `reproducibility` block (seeds, feature list, split config, deterministic flags, resolved torch/physicsnemo versions, `bitwise` claim scoped to CPU).
- **holdout-predictions parquet** ‚Äî `config_name, time, phase, wingbeat` + per-target `CF_*_true` / `CF_*_pred` (the versioned input to PR6's predicted-vs-CFD scatter).
- **`run_metadata.json`** ‚Äî via `capture_surrogate_run_metadata` (the pinned `:fp64` digest **sourced from the dataset corpus's `examples/prelim_sweep/run_metadata.json` `docker_image`** ‚Äî currently `sha256:6f971036‚Ä¶` ‚Äî git SHA, host/GPU, seeds, caller timestamp).

All four artifacts are written under **`examples/prelim_sweep/surrogate/`** (a subdirectory ‚Äî see *Implementation outcomes* D10).

### Implementation outcomes (PR5 de-risk on the A5000)

- **D10 ‚Äî artifacts in a `surrogate/` subdirectory.** The trainer's `run_metadata.json` would collide with the dataset build's `examples/prelim_sweep/run_metadata.json`; the four artifacts therefore live under `examples/prelim_sweep/surrogate/` (still "under `examples/prelim_sweep/`" per the spec). README + driver `--out-dir` updated to match.
- **D11 ‚Äî torch pinned to the cu126 index (the D5 contingency, realized).** The de-risk run confirmed PhysicsNeMo 2.1.1 stands up and `FullyConnected(in_features, layer_size, out_features, num_layers)` matches `build_model` exactly. The default PyPI `torch` is `+cu130` (CUDA 13.0), which the A5000's driver 552.86 (CUDA 12.4) rejects ("driver too old"). Pinned `torch`+`torchvision` to the linux-gated PyTorch **cu126** index (`[[tool.uv.index]]` + `[tool.uv.sources]`, `explicit = true`); cu126 (CUDA 12.6) runs on the 12.4 driver via minor-version compatibility ‚Äî `torch.cuda.is_available()` verified True on the A5000. **`torchvision` had to be added as a direct dep in the `train` group** so the cu126 source binds it (uv `[tool.uv.sources]` applies only to direct deps; the default torchvision ABI-mismatches torch+cu126: `operator torchvision::nms does not exist`).
- **D12 ‚Äî pre-PR review hardening.** The `/review-pr` subagent team (no blockers) raised quality items, fixed here: (a) the held-out-config **validation set now drives best-validation-checkpoint selection** in `train_model` and its generalization is reported under `reproducibility.val_aggregate` (previously carved but unused); (b) `compute_metrics` aggregates are uniformly **NaN-aware** (a NaN datum no longer silently nulls the headline RMSE/MAE while R¬≤ survives) and raise on a zero-row input; (c) the zero-variance guards in `Standardizer`/`compute_metrics` use an absolute floor `_VARIANCE_EPS` so a *near*-zero (not just exactly-zero) variance can't divide through to garbage; (d) `run_training` **fail-fasts** on a missing/mutable digest before training, and **warns** if a holdout config has no converged-beat rows (silent drop); (e) `_measure_inference` brackets timing with `torch.cuda.synchronize()` and averages single-row latency over 200 iters (the previous single un-synced sample was meaningless on CUDA).
- **D13 ‚Äî config-resolved (phase-honest) metrics.** A variance decomposition of the held-out predictions showed the pointwise aggregate R¬≤‚âà0.98 is **~99.9% driven by the within-beat waveform** (a smooth periodic shape shared by every config), so it overstates the kinematics‚Üíforce-*map* skill (CC-4). `metrics.json` therefore additionally reports, per target, `config_resolved.config_mean_r2` (R¬≤ on the per-config cycle-mean ‚Äî the phase-removed config-to-config skill, ‚âà0.75‚Äì0.94) and `within_config_variance_fraction` (how waveform-driven the aggregate is). The aggregate is reported **alongside**, not replaced; PR6's caption uses the honest config-resolved number.
- **Validated on hardware:** all GPU-tier tests pass on the A5000 (real CUDA), and the full real-dataset run (12,535 holdout rows, 2000 epochs) gives **held-out-config aggregate R¬≤‚âà0.98** (waveform-dominated) with **config-mean R¬≤‚âà0.75‚Äì0.94** (the honest map skill) and multi-million-row/s CUDA inference (exact numbers in the committed `surrogate/metrics.json`).

## Impact

- **Spec:** `force-surrogate` capability gains 9 requirements (held-out-config split; input/target construction; PhysicsNeMo regressor; held-out evaluation metrics; **config-resolved phase-honest metrics**; training reproducibility & provenance; committed training artifacts; two-tier cluster-free/GPU-gated tests; force-only training scope guard).
- **New code:** `src/mosquito_cfd/force_surrogate/train.py`, `scripts/train_surrogate.py`, `tests/test_force_surrogate_train.py` (CPU tier) + `tests/test_force_surrogate_train_gpu.py` (GPU tier); `__init__.py` re-exports.
